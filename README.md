# Clinic Management System ER Diagram (Database Design Assignment)

This repository contains the Entity Relationship Diagram (ERD) created as part of a database design assignment.

## Assignment Objective

The goal of this assignment is to design a database structure for a modern clinic that manages its operations digitally. The system supports managing doctors, patients, appointments, consultations, diagnostic tests, reports, and payments.

![Uploading Clinic Management System ER Diagram.png…]()

## System Description

In this clinic system:

* Patients can register and book appointments with doctors.
* Doctors conduct consultations during patient visits.
* During consultations, doctors may prescribe one or more diagnostic tests.
* Diagnostic reports are generated later and linked to the corresponding patient and consultation.
* Payments related to appointments are recorded in the system.

## Entities Included in the ER Diagram

* Patient
* Patient Family Members
* Doctors
* Appointments
* Consultations
* Diagnostic Tests
* Consultation Tests
* Reports
* Payments

## Notes

This assignment focuses only on **database modeling and ER diagram design**.
No frontend, backend, or SQL query implementation is included.

The ER diagram is provided in the repository as an exported diagram file.
